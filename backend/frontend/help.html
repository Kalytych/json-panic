📘 Кодекс JSON-оператора

Навчальний термінал для швидкого розбору JSON-синтаксису, типових помилок і правил гри

MODULE 01 · BASICS

Що таке JSON?

JSON — це текстовий формат зберігання і передачі структурованих даних. У грі він використовується як конфігурація умовних серверних систем: авторизації, бази даних, API Gateway, платежів, моніторингу та безпеки. 

{
  "service": "auth",
  "port": 8080,
  "enabled": true
}
MODULE 02 · RULES

Основні правила JSON

  • Ключі завжди пишуться у подвійних лапках.
  • Рядки також пишуться у подвійних лапках.
  • Між полями ставиться кома.
  • Після останнього поля кома не ставиться.
  • Boolean-значення: true або false.
  • null пишеться маленькими літерами.
  • Масиви записуються у квадратних дужках.
  • Об’єкти записуються у фігурних дужках.
MODULE 03 · GAME FLOW

Як працює JSON Panic?

Гравець обирає умовний сервер, відкриває пошкоджені JSON-файли й поступово відновлює стабільність серверної системи.

ERROR 01 · MISSING COMMA

Типова помилка №1: пропущена кома

Неправильно

{
  "service": "auth"
  "port": 8080
}

Правильно

{
  "service": "auth",
  "port": 8080
}
ERROR 02 · BOOLEAN CASE

Типова помилка №2: неправильний boolean

Неправильно

{
  "enabled": True
}

Правильно

{
  "enabled": true
}
ERROR 03 · WRONG TYPE

Типова помилка №3: число записане як рядок

Неправильно

{
  "port": "8080"
}

Правильно

{
  "port": 8080
}
ERROR 04 · TRAILING COMMA

Типова помилка №4: зайва кома

Неправильно

{
  "service": "auth",
  "enabled": true,
}

Правильно

{
  "service": "auth",
  "enabled": true
}
